Helpers¶

Challenge helper functions

challengeutils.helpers.archive_writeup(syn, evaluation, stat='VALIDATED', reArchive=False)[source]¶

Archive the submissions for the given evaluation queue and store them in the destination synapse folder.

Parameters

  • evaluation – a synapse evaluation queue or its ID

  • query – a query that will return the desired submissions. At least the ID must be returned. Defaults to: ‘select * from evaluation_[EVAL_ID] where status==”SCORED”’

challengeutils.helpers.create_team_wikis(syn, synid, templateid, tracker_table_synid)[source]¶

Function that creates wiki pages from a template by looking at teams that are registered for a challenge. The teams that have a wiki made for them Are stored into a trackerTable that has columns wikiSynId, and teamId

Parameters

  • synId – Synapse id of challenge project

  • templateId – Synapse id of the template

  • trackerTableSynId – Synapse id of Table that tracks if wiki pages have been made per team

challengeutils.helpers.kill_docker_submission_over_quota(syn, evaluation_id, quota=9223372036854775807)[source]¶

Kills any docker container that exceeds the run time quota Rerunning submissions will require setting TimeRemaining annotation to a positive integer

Parameters

  • syn (obj) – Synapse object

  • evaluation_id (int) – Synapse evaluation queue id

  • quota (int) – Quota in milliseconds. Default is sys.maxsize. One hour is 3600000.

challengeutils.helpers.rename_submission_files(syn, evaluationid, download_location='./', status='SCORED')[source]¶

This function renames the submission files of an evaluation queue. For many challenges we require participants to submit files that are named one thing such as prediction.csv. This function renames them to

submitter_date_filename

Parameters

  • syn – synapse object

  • evaluationid – Id of Evaluation queue

  • download_location – location to download files to (Default is ./)

  • status – The submissions to download (Default is SCORED)
